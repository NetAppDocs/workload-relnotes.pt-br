== 03 de maio de 2026



=== Amazon FSX para NetApp ONTAP



==== Clones de dispositivos de bloco

O Workload Factory oferece clones para seus dispositivos de bloco, permitindo que você provisione cópias graváveis ​​e eficientes de seus dados rapidamente, controlando a capacidade, o acesso e o ciclo de vida.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Gerencie o armazenamento em blocos no NetApp Workload Factory"]



==== Etiquetas EC2 em dispositivos de bloco

A tabela de dispositivos de bloco agora inclui uma coluna que exibe as tags dos hosts EC2 que estão consumindo cada dispositivo de bloco. Isso ajuda a identificar qual host usa cada dispositivo de bloco durante a solução de problemas ou auditorias.

Para criar tags em cada dispositivo de bloco, implante uma instância EC2 usando o script de usuário fornecido.

link:https://github.com/NetApp/FSx-ONTAP-samples-scripts/tree/main/Management-Utilities/ec2-user-data-iscsi-create-and-mount["Script de dados de usuário EC2 para atribuir tags a dispositivos de bloco"]



==== Tendências de capacidade para sistemas de arquivos FSx for ONTAP

Saiba mais sobre as tendências de capacidade do SSD diretamente na visão geral dos seus sistemas de arquivos FSx for ONTAP. O novo cartão para gerenciamento de capacidade oferece uma visão abrangente das tendências de uso do mês anterior. Essa análise inclui uma comparação entre a capacidade realmente utilizada e a capacidade alocada, permitindo uma avaliação da utilização de recursos.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-capacity-automatically.html["Gerencie a capacidade automaticamente no NetApp Workload Factory"]



=== Configuração e administração



==== Notificações por e-mail para Workload Factory

Agora você pode receber notificações do Workload Factory por e-mail sem configurar um tópico do Amazon SNS. Habilite as notificações por e-mail para enviar notificações para qualquer endereço de e-mail associado à conta.

Você pode configurar notificações por e-mail tanto no console do Workload Factory quanto no NetApp Console. O NetApp Console também oferece a opção de configurar notificações do NetApp Console que aparecem no ícone de sino.

Se você configurar as notificações no NetApp Console, NetApp Console assumirá a responsabilidade pela entrega das notificações. As configurações de notificação no console do Workload Factory se tornarão somente leitura para evitar notificações duplicadas.



==== Nova notificação para latência EDA

O serviço de notificação do NetApp Workload Factory inclui a notificação de latência de EDA, que avisa quando a latência de uma operação EDA excede um determinado limite.

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["Configurar notificações do Workload Factory"]



==== Nova notificação para sistemas de arquivos FSx for ONTAP

O serviço de notificações do NetApp Workload Factory inclui uma notificação com um resumo mensal dos custos da sua conta, mostrando tendências de gastos, alterações incomuns e as melhores oportunidades de economia para sistemas de arquivos FSx for ONTAP.

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["Configurar notificações do Workload Factory"]



=== Cargas de trabalho EDA



==== Notificações por e-mail e SNS para eventos de latência

Agora você pode receber notificações automáticas quando eventos de latência forem detectados em seus volumes FSx for ONTAP. Configure notificações por e-mail ou Amazon SNS nas configurações do Workload Factory para receber alertas em tempo real quando os volumes ultrapassarem os limites de latência configurados.

As notificações são enviadas com base no sistema de arquivos e incluem detalhes sobre todos os volumes afetados, níveis de gravidade, comparações de limites e um link direto para a página de Latência para investigação.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["Aprenda como configurar notificações de latência"^].



==== Visualização da tendência de latência com gráficos interativos

O painel de análise de latência agora inclui uma interface com abas para uma análise de latência abrangente:

* *Visão geral*: visualize os resultados da análise básica automatizada mostrando qual componente está causando latência
* *Ao longo do tempo*: Visualize o comportamento da latência de volume com um gráfico interativo em vários intervalos de tempo (1 hora, 3 horas, 12 horas, 24 horas ou 72 horas)


A aba *Ao longo do tempo* exibe dados brutos de latência do CloudWatch (leitura ou gravação, dependendo do alarme acionado) com linhas de limite de aviso e crítico, indicadores de violação e métricas detalhadas para cada violação.

Essa visualização abrangente ajuda você a identificar padrões, entender se os problemas de latência são recorrentes ou isolados e tomar decisões mais informadas sobre a correção.

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["Aprenda a analisar tendências de latência"^].

=== Amazon FSX para NetApp ONTAP



==== Atualizações de análise bem arquitetadas

O Workload Factory analisa seus sistemas de arquivos FSx for ONTAP em busca de *volumes NAS inativos*. A análise de configuração identifica volumes NAS que não estão sendo usados ativamente e recomenda ações para otimizar a utilização do storage e reduzir custos.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Implementar FSx bem arquitetado para sistemas de arquivos ONTAP"]



==== Funcionalidade de tabela de diário para pontos de acesso S3

O recurso de tabela de diário oferece uma maneira de gerenciar FSx ONTAP S3 storage de objetos em escala, permitindo auditar facilmente os eventos de acesso do usuário e as operações em objetos nos seus buckets. Com o Workload Factory, você pode configurar varreduras automáticas dos seus buckets e tabelas S3 para manter um registro de eventos, mostrando todos os objetos, seus detalhes e as ações do usuário.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["Configure a infraestrutura da tabela de diários para NetApp Workload Factory"]



==== Suporte para gerenciamento de igroups existentes

O Workload Factory oferece suporte ao gerenciamento de igroups existentes. Você pode adicionar ou remover dispositivos de bloco e iniciadores de host para entregar sua carga de trabalho implantada no prazo.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Gerencie o armazenamento em blocos no NetApp Workload Factory"]



==== Suporte para especificar o agregado para volumes FlexVol

O Workload Factory permite criar volumes FlexVol em um agregado escolhido. Isso possibilita dividir a carga de capacidade e desempenho entre todos os recursos do sistema de arquivos.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["Crie um FlexVol volume no NetApp Workload Factory"]



==== Último tempo de acesso para dispositivos de bloco

O Workload Factory registra o último tempo de acesso aos dispositivos de bloco. Esse tempo é baseado no último acesso ao igroup no Workload Factory. Saber quando cada dispositivo de bloco foi usado pela última vez permite otimizar seus recursos.



=== Workloads de banco de dados



==== Análise bem arquitetada para Oracle

A análise bem arquitetada inclui as seguintes novas configurações para Oracle:

* *Atualizações Críticas de Segurança da Oracle*: avalia a ausência de patches de segurança para instalações de banco de dados Oracle autogerenciadas suportadas.
* *Replicação entre regiões*: verifica se a replicação entre regiões (CRR) está habilitada para seus sistemas de arquivos FSx for ONTAP que atendem a bancos de dados Oracle.
* *Snapshots consistentes com a aplicação*: verifica se o banco de dados Oracle está protegido por snapshots consistentes com a aplicação NetApp SnapCenter.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementar configurações de banco de dados bem arquitetadas no Workload Factory"]



==== Avaliação única para Oracle

O Workload Factory agora oferece uma avaliação única para analisar as configurações de suas implementações de banco de dados Oracle sempre que necessário — novas implementações, pós-migração, solução de problemas e muito mais. Você receberá recomendações para ajudar a otimizar suas configurações de implementação e a cumprir as melhores práticas. Esta é a sua oportunidade de usar o recurso well-architected sem precisar se comprometer totalmente com o processo de configuração. Você pode avaliar seus bancos de dados sem armazenar credenciais no Workload Factory, registrar recursos de banco de dados ou configurar conexões externas.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Realize uma avaliação única das suas implementações de banco de dados Oracle"]



==== Melhoria na redução de custos em diversos bancos de dados Oracle

Aprimoramos a análise de redução de custos para Oracle para fornecer uma solução de storage consolidada para vários bancos de dados Oracle locais em um único sistema de arquivos FSx for ONTAP. Essa consolidação otimiza os custos de armazenamento ao reduzir o número de sistemas de arquivos necessários para vários bancos de dados, resultando em maior economia.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Explore as economias para bancos de dados detectados em Oracle on-premises"]

=== Cargas de trabalho EDA



==== Recomendações de capacidade EDA

O Workload Factory agora analisa automaticamente a capacidade SSD de cada FSx for ONTAP cluster em seu inventário EDA e gera recomendações de capacidade quando ajustes são necessários. O algoritmo é executado uma vez a cada 24 horas para cada cluster.

Quando uma recomendação for identificada:

* Você recebe um e-mail ou notificação WAD imediatamente (não faz parte do resumo semanal).
* Um indicador em forma de lâmpada aparece no cluster afetado na tabela Clusters.
* Um resumo no painel de Visão Geral mostra o número total de clusters com recomendações ativas.


Selecione um cluster com uma recomendação ativa para visualizar a recomendação, que explica a alteração sugerida e o raciocínio.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["Saiba mais sobre as recomendações de capacidade"^].



==== Análise automatizada de latência de IA

A análise de latência agora inclui a análise de agente de IA.

Para cenários complexos que envolvem dados ou latência de cluster, você pode invocar a análise de agentes de IA para receber:

* Identificação detalhada da causa raiz
* Lista de clientes EC2 afetados
* Etapas específicas de remediação


A análise com agente de IA requer a configuração do ARN do modelo Amazon Bedrock e está disponível sob demanda para cenários em que a análise básica recomenda uma investigação mais aprofundada. O agente de IA pode identificar problemas como volumes excessivos, configurações de ponto de montagem não ideais, necessidades de rebalanceamento de FlexGroup e requisitos de escalabilidade horizontal com escalabilidade horizontal.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["Saiba mais sobre análise de agentes de IA"^].

=== Workloads da VMware



==== Apresentando o suporte do migration advisor para Google Cloud VMware Engine

NetApp Workload Factory agora é compatível com o Google Cloud VMware Engine. Você pode migrar suas cargas de trabalho VMware locais para o Google Cloud VMware Engine usando o Google Cloud NetApp Volumes como datastores NFS externos, otimizando custos e permitindo maior controle sobre seu ambiente VMware sem a necessidade de refatorar ou replataformar seus aplicativos.

O consultor de migração do Google Cloud VMware Engine fornece:

* Dimensionamento e design automatizados de clusters do Google Cloud VMware Engine com suporte para tipos de nó VE1 e VE2
* Otimização do layout de máquinas virtuais entre datastores internos do vSAN e datastores NFS externos do Google Cloud NetApp Volumes.
* Comparação detalhada de custos e análise de TCO entre configurações somente vSAN e configurações aprimoradas pelo Google Cloud NetApp Volumes.
* Suporte para os níveis de serviço Standard, Premium e Extreme do Google Cloud NetApp Volumes.
* Implantação de Infrastructure-as-Code com Terraform para provisionamento automatizado.

=== Amazon FSX para NetApp ONTAP



==== Atualizações de análise bem arquitetadas

O Workload Factory analisa seus sistemas de arquivos FSx para ONTAP para as seguintes configurações:

* Capacidade SSD subutilizada: verifica se a camada de capacidade SSD está subutilizada e recomenda diminuir a capacidade SSD para otimizar custos.
* Agende backups de volumes: O NetApp Workload Factory aprimora a recomendação de configuração e adiciona opções para ativar o FSx for ONTAP backup ou o AWS Backup. FSx for ONTAP backup aplica uma política padrão a todos os volumes em um sistema de arquivos, criando backups diários mantidos por até 90 dias. AWS Backup permite escolher diferentes períodos de retenção para cada volume no plano de backup.




==== O Workload Factory oferece suporte ao caso de uso de IA e ML para replicação

O Workload Factory oferece suporte à replicação de dados ONTAP locais para o FSx for ONTAP e à criação de pontos de acesso S3 para serviços de Artificial Intelligence (AI) e machine learning (ML) da AWS, a fim de acelerar os fluxos de trabalho de AI/ML.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["Replicar dados locais para casos de uso de AI  ML"]



==== Editando pontos de acesso S3 existentes

Você pode alterar o usuário e o tipo de usuário de um ponto de acesso S3 existente conectado a um volume. Você também pode ativar ou desativar metadados para o ponto de acesso.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/edit-s3-access-points.html["Editar pontos de acesso S3 no NetApp Workload Factory"]



==== Diagrama de armazenamento em bloco adicionado

Se você estiver usando block storage, poderá visualizar o diagrama de block storage para visualizar como os block devices se conectam aos nodes e igroups. O diagrama ajuda você a entender e gerenciar seu block storage.

==== Atualização no recurso de gerenciamento automático de capacidade

Esta atualização separa a funcionalidade em duas funcionalidades distintas: gerenciamento de capacidade automático e notificações de capacidade.

O gerenciamento de capacidade adiciona ou remove capacidade do sistema de arquivos automaticamente para manter o desempenho ideal. Novas opções incluem os modos storage incremental e adaptável. No modo storage incremental, o sistema aumenta a capacidade em uma quantidade fixa em determinados limites; no modo adaptável, ele usa dados históricos para prever e ajustar a capacidade.

As notificações de capacidade alertam você sempre que ocorre um ajuste de capacidade e quando os sistemas de arquivos atingem os limites definidos, para que você possa gerenciar storage antes que surjam problemas.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/enable-auto-capacity-management.html["Aprenda sobre gerenciamento automático de capacidade e notificações de capacidade no NetApp Workload Factory"]

=== Amazon FSX para NetApp ONTAP



==== Suporte para migração de storage VM

NetApp Workload Factory agora oferece suporte à migração para VMs de armazenamento. Esse recurso permite a migração de dados e configurações do sistema de storage ONTAP de sistemas ONTAP locais ou sistemas de arquivos FSx for ONTAP de primeira geração para sistemas de arquivos FSx for ONTAP de segunda geração. Você pode replicar os dados e as configurações da VM de armazenamento para migrar para o novo sistema de arquivos com tempo de inatividade e interrupção mínimos para usuários e aplicativos.

Para usar este recurso, link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["criar uma relação de replicação"] e selecione *Migração* como caso de uso. Para concluir o processo de migração, você deve link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html#cut-over-replication-for-migration-use-cases["migrar o storage VM e seus volumes replicados"] imediatamente para migrar permanentemente os dados e as configurações da VM de storage para o sistema de arquivos FSx for ONTAP de destino.
