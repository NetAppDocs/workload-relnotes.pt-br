=== Workloads da VMware



==== Apresentando o suporte do migration advisor para Google Cloud VMware Engine

NetApp Workload Factory agora é compatível com o Google Cloud VMware Engine. Você pode migrar suas cargas de trabalho VMware locais para o Google Cloud VMware Engine usando o Google Cloud NetApp Volumes como datastores NFS externos, otimizando custos e permitindo maior controle sobre seu ambiente VMware sem a necessidade de refatorar ou replataformar seus aplicativos.

O consultor de migração do Google Cloud VMware Engine fornece:

* Dimensionamento e design automatizados de clusters do Google Cloud VMware Engine com suporte para tipos de nó VE1 e VE2
* Otimização do layout de máquinas virtuais entre datastores internos do vSAN e datastores NFS externos do Google Cloud NetApp Volumes.
* Comparação detalhada de custos e análise de TCO entre configurações somente vSAN e configurações aprimoradas pelo Google Cloud NetApp Volumes.
* Suporte para os níveis de serviço Standard, Premium e Extreme do Google Cloud NetApp Volumes.
* Implantação de Infrastructure-as-Code com Terraform para provisionamento automatizado.

=== Workloads de banco de dados



==== Avaliação única

Workload Factory agora oferece uma avaliação única para analisar as configurações de suas implantações de banco de dados sempre que necessário - novas implantações, pós-migração, solução de problemas e muito mais. Você receberá recomendações para ajudar a otimizar suas configurações de implantação e cumprir as melhores práticas. Esta é a sua oportunidade de usar o recurso well-architected sem se comprometer totalmente com o processo de configuração. Você pode avaliar seus bancos de dados sem armazenar credenciais no Workload Factory, registrar recursos de banco de dados ou configurar conexões externas.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Realize uma avaliação única das suas implementações de banco de dados"]



==== Suporte para registro de grupos de disponibilidade Always On no Microsoft SQL Server e Data Guard no Oracle

Agora você pode descobrir e registrar instâncias do Microsoft SQL Server com o modelo de implantação Always On availability group (AOAG) e bancos de dados Oracle com o modelo de implantação Data Guard na carga de trabalho de bancos de dados do NetApp Workload Factory. Essa melhoria permite um gerenciamento mais eficiente dos recursos do SQL Server e do Oracle no console do Workload Factory.

Após registrar suas instâncias do AOAG e bancos de dados do Data Guard, você pode aproveitar os diversos recursos do NetApp Workload Factory for Databases, como visualizar detalhes dos recursos e analisar logs de erros.

Workload Factory for Databases também oferece suporte à análise bem arquitetada para instâncias do Microsoft SQL Server com o modelo de implantação AOAG. Você pode analisar as configurações de suas instâncias do SQL Server em um grupo de disponibilidade e receber recomendações para otimizar suas configurações de implantação e cumprir as melhores práticas. Registre suas instâncias AOAG para habilitar esse recurso.

link:https://docs.netapp.com/us-en/workload-databases/register-instance.html#register-a-sql-server-instance["Registre recursos no Workload Factory for Databases"]



==== Análise bem arquitetada para Oracle

A análise bem arquitetada inclui *patches do sistema operacional* para Oracle para verificar se você possui os patches mais recentes no sistema operacional e informar como instalá-los.

Além disso, a análise bem arquitetada para Oracle notifica você quando uma configuração, como layout de storage, tem dependências de outras configurações. A notificação informa quais ações tomar para otimizar as configurações vinculadas, de modo que todas as recomendações que fornecemos sejam ideais.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementar configurações de banco de dados bem arquitetadas no Workload Factory"]



==== Análise de custos e planejamento de migração de ambientes de banco de dados Oracle on-premises

O Workload Factory for Databases agora analisa e ajuda você a planejar a migração de banco de dados Oracle no local para Amazon FSx for NetApp ONTAP. Você pode usar a calculadora de economia para estimar o custo de execução do seu ambiente de banco de dados Oracle no local na nuvem e consultar recomendações para migrar seu ambiente de banco de dados no local para a nuvem.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html["Explore as oportunidades de economia em ambientes de banco de dados locais"]

=== Amazon FSX para NetApp ONTAP



==== Atualizações de análise bem arquitetadas

O Workload Factory analisa seus sistemas de arquivos FSx para ONTAP para as seguintes configurações:

* Capacidade SSD subutilizada: verifica se a camada de capacidade SSD está subutilizada e recomenda diminuir a capacidade SSD para otimizar custos.
* Agende backups de volumes: O NetApp Workload Factory aprimora a recomendação de configuração e adiciona opções para ativar o FSx for ONTAP backup ou o AWS Backup. FSx for ONTAP backup aplica uma política padrão a todos os volumes em um sistema de arquivos, criando backups diários mantidos por até 90 dias. AWS Backup permite escolher diferentes períodos de retenção para cada volume no plano de backup.




==== O Workload Factory oferece suporte ao caso de uso de IA e ML para replicação

O Workload Factory oferece suporte à replicação de dados ONTAP locais para o FSx for ONTAP e à criação de pontos de acesso S3 para serviços de Artificial Intelligence (AI) e machine learning (ML) da AWS, a fim de acelerar os fluxos de trabalho de AI/ML.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["Replicar dados locais para casos de uso de AI  ML"]



==== Editando pontos de acesso S3 existentes

Você pode alterar o usuário e o tipo de usuário de um ponto de acesso S3 existente conectado a um volume. Você também pode ativar ou desativar metadados para o ponto de acesso.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-s3-access-points.html["Editar pontos de acesso S3 no NetApp Workload Factory"]



==== Diagrama de armazenamento em bloco adicionado

Se você estiver usando block storage, poderá visualizar o diagrama de block storage para visualizar como os block devices se conectam aos nodes e igroups. O diagrama ajuda você a entender e gerenciar seu block storage.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Gerencie o armazenamento em blocos no NetApp Workload Factory"]



==== Atualização no recurso de gerenciamento automático de capacidade

Esta atualização separa a funcionalidade em duas funcionalidades distintas: gerenciamento de capacidade automático e notificações de capacidade.

O gerenciamento de capacidade adiciona ou remove capacidade do sistema de arquivos automaticamente para manter o desempenho ideal. Novas opções incluem os modos storage incremental e adaptável. No modo storage incremental, o sistema aumenta a capacidade em uma quantidade fixa em determinados limites; no modo adaptável, ele usa dados históricos para prever e ajustar a capacidade.

As notificações de capacidade alertam você sempre que ocorre um ajuste de capacidade e quando os sistemas de arquivos atingem os limites definidos, para que você possa gerenciar storage antes que surjam problemas.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/enable-auto-capacity-management.html["Aprenda sobre gerenciamento automático de capacidade e notificações de capacidade no NetApp Workload Factory"]

Como administrador de TI responsável por dezenas de FSx for ONTAP sistemas de arquivos, você pode usar o painel de visão geral para responder a perguntas operacionais comuns sobre a integridade do cluster, a utilização e o posicionamento de volumes.

O painel principal anterior agora está acessível na aba *Project*. As abas de EDA agora estão organizadas da seguinte forma: Overview, Latency, Project, CI/CD.

A análise básica avalia as políticas de QoS, o desempenho do disco, os problemas de rede e as configurações do cluster para fornecer informações imediatas na coluna Detalhes da tabela de eventos de latência.

==== Redução de custos aprimorada para vários hosts de banco de dados no Microsoft SQL Server no local

A análise de redução de custos para Microsoft SQL Server storage no local foi aprimorada para fornecer uma solução de storage consolidada para vários hosts de banco de dados em um sistema de arquivos FSx for ONTAP. Essa consolidação otimiza os custos de storage ao reduzir o número de sistemas de arquivos necessários para vários hosts de banco de dados, levando a uma maior economia de custos.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Explore as economias para hosts detectados no Microsoft SQL Server no local"]



==== Análise bem arquitetada para Oracle

A análise bem arquitetada inclui quatro novas subavaliações para a análise da configuração de storage do Oracle: capacitação do dNFS, resolução de IP consistente do dNFS, arquivo de configuração do dNFS e dNFS `nosharecache`. Essas avaliações se relacionam à habilitação e configuração do Direct NFS (dNFS) para o seu ambiente Oracle, que ignora o cliente NFS e executa operações de arquivo NFS diretamente em um servidor NFS, melhorando o desempenho de E/S e diminuindo a carga no host e no sistema de storage porque a E/S é realizada de forma mais eficiente.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementar configurações de banco de dados bem arquitetadas no Workload Factory"]

==== Atualizações de análise bem arquitetadas

O Workload Factory analisa seus sistemas de arquivos FSx para ONTAP para as seguintes configurações:

* Otimizar o tamanho do volume de cache: verifica se o dimensionamento automático e a limpeza estão ativados nos volumes de cache para manter o tamanho ideal e concentrar o cache nos dados mais acessados para eficiência máxima.
* Dispositivos de bloco inativos: recomenda arquivar os dados do dispositivo de bloco ou excluir um dispositivo de bloco se ele não for usado por sete dias consecutivos.
* Relatório lógico de storage VM: verifica se a configuração padrão de relatório para uma storage VM está definida como lógica, o que proporciona melhor visibilidade do uso de storage no nível do volume.




==== Placas adicionais para dispositivos de bloco

Três novos cartões foram adicionados à guia Dispositivos de bloco na carga de trabalho de storage para fornecer informações rápidas sobre o uso do dispositivo de bloco e o status de proteção:

* Eficiências de armazenamento: exibe a capacidade de storage usada e disponível; a capacidade usada é discriminada por SSD e por níveis de storage do pool de capacidade.
* Dispositivos protegidos: exibe a porcentagem de dispositivos de bloco com snapshots, replicação remota, NetApp Autonomous Ransomware Protection (ARP/AI) e backups.
* Dispositivos órfãos: exibe se algum dispositivo de bloco não foi usado por sete dias consecutivos, ajudando você a identificar e gerenciar recursos não utilizados com eficiência. A partir daqui, você pode link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html#archive-the-data-of-an-inactive-block-device["recuperar espaço para dispositivos de bloco não utilizados"].




==== Suporte para criação e gerenciamento de grupos de iniciadores

NetApp Workload Factory oferece suporte à criação e ao gerenciamento de grupos de iniciadores (igroups) para armazenamento em bloco em FSx for ONTAP sistemas de arquivos. Os grupos de iniciadores conectam dispositivos de bloco (LUNs) aos recursos de computação que têm permissão para acessá-los, fornecendo uma camada de permissões para armazenamento em bloco em ambientes SAN.

* link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-igroup.html["Crie um grupo de iniciadores no NetApp Workload Factory"]

=== Workloads da VMware



==== Análise bem arquitetada para Amazon Elastic VMware Service

O Workload Factory agora oferece análise automatizada bem arquitetada para seus ambientes do Amazon Elastic VMware Service (Exchange Virtual Server). Varreduras diárias identificam desalinhamentos de configuração e oferecem recomendações detalhadas de correção para ajudar você a manter operações, segurança e eficiência de custos ideais.

As análises são realizadas usando APIs da AWS—sem necessidade de credenciais vSphere ou conectividade vCenter necessárias. Os resultados estão disponíveis na nova guia *Well-architected status* nos detalhes do seu ambiente Exchange Virtual Server.

Esta versão inclui insights para:

* *Status de proteção contra parada/encerramento de instância*: Identifica nós Exchange Virtual Server sem proteção contra parada ou encerramento do EC2. Parar ou encerrar nós Exchange Virtual Server a partir do console do EC2 pode levar à indisponibilidade de dados da máquina virtual ou perda de dados.
* *Alinhamento do posicionamento de partições de nós de cluster*: Detecta desalinhamentos de particionamento que podem resultar em perda significativa de poder de processamento ou tempo de inatividade caso uma partição falhe em uma Zona de Disponibilidade da AWS. Cada insight inclui níveis de gravidade, resultados detalhados com informações sobre os recursos afetados e procedimentos de correção passo a passo com base nas melhores práticas da AWS.


link:view-evs-well-architected.html["Implementar configurações bem arquitetadas"]

=== Cargas de trabalho EDA



==== Filtragem aprimorada do painel com tags personalizáveis

Agora você pode configurar até cinco filtros personalizados no painel do EDA com base em suas tags da AWS. Cada filtro personalizado inclui um nome de rótulo, uma chave de tag da AWS e um tipo de seleção (seleção única ou multi-seleção).

A seleção múltipla permite selecionar vários valores simultaneamente, enquanto a seleção única restringe você a um valor por vez. Os filtros personalizados aparecem na ordem em que você os configura, facilitando a organização dos filtros mais usados.

Se você não configurar filtros personalizados, os filtros padrão (sistema de arquivos, tipo de volume e intervalo de tempo) permanecerão disponíveis para que você possa continuar visualizando e interagindo com seus dashboards.

https://docs.netapp.com/us-en/workload-eda/task-configure-dashboard.html["Saiba mais sobre configurar filtros personalizados"^].



==== Visualização detalhada do volume para análise granular de desempenho

O painel agora oferece dois modos de visualização: visualização total e visualização por volume. A visualização total exibe métricas agregadas de todos os volumes, enquanto a visualização por volume mostra o desempenho individual dos 10 principais volumes ao longo do tempo.

Na visualização de Volume, dicas de ferramentas interativas ao passar o cursor fornecem informações detalhadas para cada volume, incluindo nome do volume, métricas e valores específicos de tempo. Quando os mesmos volumes aparecem em vários componentes, a codificação por cores consistente facilita o rastreamento de volumes específicos em diferentes métricas.

https://docs.netapp.com/us-en/workload-eda/task-configure-dashboard.html#view-volume-details["Saiba mais sobre como visualizar detalhes do volume"^].



==== Análise de latência para monitoramento proativo de desempenho

A análise de latência permite monitorar a latência de leitura e gravação de volumes em seus sistemas de arquivos FSx for ONTAP. Você pode configurar limites de aviso e eventos críticos personalizáveis para identificar proativamente gargalos de desempenho antes que eles impactem suas cargas de trabalho de EDA.

A tabela de eventos de latência exibe todos os eventos de aviso e críticos, permitindo que você monitore o desempenho do volume e identifique os volumes que precisam de otimização.

Esse recurso requer credenciais da AWS e é acessível no menu Latency no painel EDA.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html["Saiba mais sobre análise de latência"^].
